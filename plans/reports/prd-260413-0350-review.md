## Next Feature Recommendation: Expense Categories (S)

Rationale: smallest effort, zero algorithmic risk, unlocks future analytics/filter work, closes a PRD gap in epic-03, no dependency on other pending features. Edit Bill deferred — needs UX for split recalculation (more thought than one cycle).

### User Story
> As a group member, I want to tag each bill with a category (Ăn uống, Đi lại, Lưu trú, Mua sắm, Giải trí, Khác) so I can quickly see what the group spends money on.

### Acceptance Criteria
- `bills.category` column added, nullable text, default `'khac'`, check constraint on 6 allowed values
- AI parser (`parse-bill` API) infers category from title keywords; falls back to `'khac'`
- Bill Confirm Sheet (US-E3-2) shows category chip, tappable → bottom sheet picker with 6 options + emoji
- Bill Card in chat shows category emoji prefix on title (e.g. "🍜 Cafe 180k")
- Open Bill card shows category chip below title
- Existing bills migrated as `'khac'`
- Edit category available via same 3-dot menu → "Đổi danh mục" (simple, no full edit-bill needed)
- No new tab/filter UI this cycle (YAGNI — add when analytics ships)
