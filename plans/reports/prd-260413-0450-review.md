## 3. Mini-Spec: Simplify Debts (Pairwise Netting v1)

### Algorithm (KISS — pairwise only for v1)
```
For each (debtor, creditor) pair in group's pending debts:
  sum_ab = sum of remaining where debtor=A, creditor=B
  sum_ba = sum of remaining where debtor=B, creditor=A
  net = sum_ab - sum_ba
  if net > 0: A owes B net
  if net < 0: B owes A |net|
  if net == 0: cancelled
```
**NOT** graph-based multi-hop (A→B→C becomes A→C). Deferred to v2. Pairwise alone handles the common case (person paid for you yesterday, you paid for them today).

### Acceptance Criteria
1. Debts page shows **netted view by default** — grouped by (you ↔ other person) pair, displaying single net amount per pair
2. Toggle "Xem chi tiết" reveals underlying per-bill debts
3. Settlement (mark as paid) operates on **net amount** — marks all underlying pending debts between that pair as paid atomically
4. Group detail card subtitle (US-E2-1) uses netted totals
5. Zero-net pairs (A paid 100k for B, B paid 100k for A) show as "Đã cân" / hidden
6. Works in realtime: debts channel subscription re-computes netting
