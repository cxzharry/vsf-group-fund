## Next Feature: Edit Bill (S)

**Why:** completes cycle 3 stub, low risk, high trust (fix typos), simpler than simplify-debts.

## Spec

### Entry
- 3-dot menu on bill card → existing "Sửa" stub → open EditBillSheet
- Only visible if `bill.paid_by === currentMember.id`

### Preconditions (guards)
- Bill owner only (paid_by match)
- Block if any related `debts` row has `payment_confirmation != null` OR `status = 'paid'` → toast "Không thể sửa: đã có thanh toán"
- Block if bill is part of simplified-debts snapshot (check `simplified_debt_id` on debts) → toast "Không thể sửa: nợ đã gộp"

### Editable fields (v1)
- `description` (text, 1–100 chars)
- `amount` (positive int VND, token format on input)
- Category (optional, reuse cycle 4 picker)

### Locked fields (v1)
- Split mode, participants, paid_by, created_at — too risky to recompute membership

### Save flow (transaction)
1. Validate amount > 0, desc non-empty
2. Re-run existing split logic with new amount + same participants + same mode → new per-member shares
3. UPDATE `bills` SET description, amount, category_id, updated_at
4. For each related `debts` row:
   - delta = new_share - old_share
   - UPDATE debts SET amount = new_share, remaining = remaining + delta
   - If remaining ≤ 0 → status = 'paid' (edge: amount reduced below already-paid portion)
5. Insert audit row in `bill_edits` (bill_id, editor_id, old_amount, new_amount, old_desc, new_desc, edited_at)
6. revalidatePath group detail

### UI
- Reuse AddBillSheet via `mode: 'edit' | 'create'` prop (DRY)
- Header: "Sửa bill" / Button: "Lưu thay đổi"
- Subtle "Đã sửa" badge on bills with bill_edits rows

### DB
- New table `bill_edits` (id, bill_id FK, editor_id FK, old_amount, new_amount, old_desc, new_desc, edited_at)
- RLS: group membership via bill_id join
