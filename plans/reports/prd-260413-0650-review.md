## Current State
`src/lib/simplify-debts.ts` only performs **pairwise netting** (A↔B cancellation). Misses A→B→C→A cycles and transitive reductions. Existing interface `SimplifiedDebt` already supports multi-underlying-id merging.

## Algorithm: Greedy Balance Settlement
1. Build net balance map: `balance[user] = sum(credits) - sum(debts)`
2. Partition into creditors (+) and debtors (−)
3. Repeatedly match largest creditor with largest debtor, emit payment `min(|debt|, credit)`, update balances
4. Terminate when all balances within epsilon (1 VND)

**Properties:**
- Produces ≤ N−1 payments for N participants (provably minimal in worst case)
- O(N log N) per iteration via sorted structures; fine for N<50
- Loses per-pair traceability → `underlying_ids` becomes union of all debts in the connected component

## Mini-Spec
- **New export:** `simplifyDebtsGraph(debts: RawDebt[]): SimplifiedDebt[]`
- **Keep** `simplifyDebts` (pairwise) for backward compat
- **Wire-up:** debts page "Nợ ròng" mode calls `simplifyDebtsGraph` (strictly better, no toggle needed — KISS)
- **Edge cases:** empty input, single debt, balanced cycle (all zero), floating-point rounding (use integer VND)
